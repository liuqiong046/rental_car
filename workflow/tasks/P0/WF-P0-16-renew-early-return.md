## 目标

实现租赁中订单的续租申请、提前还车申请、审核和关联任务/费用同步。

## 业务闭环

用户在 C 端发起续租或提前还车，PC 后台或授权管理端审核后，同步更新客户订单、批发订单、还车工单和费用待处理事项。

## 输入文档

- PRD §5.11、§7.7、§7.8、§13

## 范围内改动

- C 端续租和提前还车入口。
- 审核接口和状态展示。
- 工单时间、批发订单和费用同步。

## 非目标

- 不实现复杂审批流。
- 不实现高级节假日规则之外的二期策略。

## 前置依赖

- `WF-P0-15`

## 交付物

- 续租/提前还车申请。
- 审核通过/拒绝。
- 关联任务和费用同步记录。

## 验收动作

1. 租赁中订单可发起续租和提前还车。
2. 审核拒绝原因同步 C 端。
3. 审核通过后关联批发订单和工单生成待处理提醒或同步变更。

## 完成定义

- 后续售后和财务可基于最终任务和费用进入结算。
