# 山海放心租三端首期技术栈方案 V1

## Context

本方案基于 `doc/requirements/shanhai-rental-mini-programs-requirements.md` 制定。首期目标是围绕租车交易、批发调车、移动履约与 PC 主控后台建立可研发、可验收的三端协同系统。

当前仓库已有 C 端微信原生小程序页面，已整理到 `miniprogram/`，可作为 C 端首期落地基础。后端、管理端小程序、PC 管理后台和部署工程按本方案新增骨架。

## 1. 总体架构

```text
C 端微信小程序        管理端微信小程序       PC 管理后台
找车/认证/下单/支付   待办/排车/工单/车况    配置/审核/订单/财务/审计
        \                  |                  /
         \                 |                 /
          ---------- HTTPS JSON API ----------
                         |
                    FastAPI API
        auth / user / vehicle / price / order / wholesale
        work-order / payment / finance / admin / message
                         |
             MySQL 8 + Redis 7 + Arq Worker
                         |
     微信登录/支付、腾讯地图、对象存储、订阅消息、短信预留
```

部署单元建议为 `api`、`worker`、`mysql`、`redis`、`nginx`、`admin-web`。两个小程序由微信开发者工具和微信平台发布。

## 2. 已锁定选型

| 层 | 选型 | 理由 |
| --- | --- | --- |
| 后端语言 | Python 3.12 | 与 onePet 基线一致，适合快速建立业务 API 与异步任务 |
| 后端框架 | FastAPI + Uvicorn | 自动 OpenAPI、Pydantic DTO、适合三端 API 协作 |
| ORM / 迁移 | SQLAlchemy 2.0 async + Alembic | 明确事务边界，支持订单、支付、退款、结算等复杂状态 |
| 校验 / DTO | Pydantic v2 | 请求响应模型可直接驱动接口文档和测试 |
| 关系库 | MySQL 8 | 订单、车辆、价格、财务、权限与审计主库 |
| 缓存 / 锁 | Redis 7 | 未支付锁车、确认 SLA、幂等键、会话、热点配置 |
| 异步任务 | Arq | 未支付超时关单、车行确认 SLA、通知、退款轮询 |
| C 端 | 当前微信原生小程序 | 已有页面与视觉资产，首期不强制迁移 TypeScript |
| 管理端小程序 | 微信原生小程序 | 移动履约工具，贴近微信运行环境 |
| PC 后台 | Vue 3 + TypeScript + Vite + Element Plus | 适合表格、表单、审核、配置、财务后台 |

## 3. 后端组件矩阵

| 组件 | 选型 | 用途 |
| --- | --- | --- |
| API 分层 | `router.py` / `schemas.py` / `service.py` / `deps.py` | 参考 onePet，保持路由、DTO、业务编排、依赖隔离 |
| 配置 | pydantic-settings | 环境变量、第三方密钥、运行环境配置 |
| 鉴权 | JWT access token + Redis 黑名单/续签 | C 端、管理端、PC 后台统一登录态 |
| 权限 | Casbin 或等价 RBAC | 菜单、按钮、基础数据范围和角色权限 |
| 幂等 | `Idempotency-Key` + Redis/MySQL 记录 | 下单、支付、退款、接单、批发接拒单、费用调整 |
| 状态机 | 显式 Enum + 迁移白名单 | 客户订单、批发订单、工单、退款状态 |
| 定时任务 | Arq deferred jobs | 未支付关单、批发确认超时、通知提醒、退款轮询 |
| 对象存储 | 腾讯云 COS 或兼容对象存储 | 身份证、驾驶证、车辆、车况、凭证图片 |
| 地图能力 | 腾讯地图服务 | 定位、地址选择、距离、服务半径、超区费 |
| 安全 | AES-GCM 字段加密 + 日志脱敏 | 手机号、身份证、驾驶证、详细地址、支付配置 |
| 审计 | `audit_log` 表 + 装饰器/服务封装 | 金额、退款、结算、权限、导出、敏感信息查看 |

## 4. 前端组件矩阵

### 4.1 C 端微信小程序

- 沿用 `miniprogram/` 下的微信原生小程序结构：`pages/`、`utils/`、`assets/`、`app.json`、`app.wxss`。
- 视觉基线沿用 README：主色 `#FCD41D`，背景 `#F8F8F7`，主要卡片宽度 `686rpx`。
- 新增接口封装时优先集中在 `utils/request.js` 或后续任务约定的 `services/`。
- 首期页面围绕首页、品牌、车辆列表、详情、确认订单、订单、我的、登录继续迭代。

### 4.2 管理端微信小程序

- 新建 `manager-miniprogram/`。
- 首期聚焦移动履约，不承接完整后台配置。
- 页面模块：待办、控制台、客户订单、批发订单、调车、工单、车况录入、现场费用、我的任务。
- 不支持离线提交车况照片，无网络时提示恢复网络后提交。

### 4.3 PC 管理后台

- 新建 `admin-web/`。
- 使用 Vue 3、TypeScript、Vite、Element Plus、Pinia、Vue Router。
- 模块：工作台、城市配置、组织权限、车辆库存、价格日历、身份审核、客户订单、批发订单、工单排班、财务结算、售后投诉、通知日志、固定导出。
- 表格页统一封装筛选、分页、列展示、导出入口和权限按钮。

## 6. 关键技术决策

1. 未支付订单短时锁车：创建待支付订单时写入锁车到期时间，并用 Redis 记录车辆时间段锁；Arq 到期后关闭订单并释放车辆。
2. 价格快照：用户提交订单时锁定租金、服务费、押金提示、价格日历版本和计费明细，后续价格变更不影响已支付订单。
3. 车行确认 SLA：关联批发订单创建后按城市配置投递超时任务，超时自动拒绝并同步客户订单进入待改派/待退单。
4. 改派差价：高价车差额记运营中心让利或人工调整；低价车按新价重算并原路退差或进入待结算退差。
5. 押金/免押：首期后置到取车前，由 PC 后台或授权管理端人工确认；未确认时取车工单不可完成。
6. 退款与结算：退款必须由 PC 后台发起并经财务岗审核，支付流水、退款流水、人工调整和批发成本都要留痕。
7. 数据范围：总部、运营中心、车行、司机的数据范围必须在服务端隔离。
8. 敏感数据：手机号、身份证、驾驶证、详细地址默认脱敏展示；查看明文需权限和审计。
9. 导出：首期只做固定模板同步导出，导出必须校验权限并记录审计日志。
10. API 文档：FastAPI OpenAPI 是三端接口协作主入口，接口变更必须同步 `doc/接口规范-V1.md` 或任务包契约说明。
