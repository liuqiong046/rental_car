# 山海放心租三端首期产品需求文档

版本：v1.0  
日期：2026-05-06  
适用产品：山海放心租 C 端小程序、山海管家管理端小程序、山海放心租 PC 管理后台

## 1. 文档目标

本文档用于将已有 UI 设计、功能清单和业务讨论结果收敛为首期可研发、可排期、可验收的产品需求。首期不是全量平台化版本，而是围绕租车交易和履约闭环，完成三端协同的最小可运营版本。

首期产品边界：

1. C 端小程序负责用户找车、身份认证、下单、支付租金、查看订单和确认售后。
2. 管理端小程序负责移动履约，包括待办、排车、工单执行、车况录入、现场费用和批发订单移动处理。
3. PC 管理后台负责配置、审核、订单主控、车辆与价格、规则、财务结算、权限和操作日志。
4. 首期保留批发调车和客户订单关联批发订单闭环。
5. 渠道收益、渠道价格助手、托管个人完整端、复杂审批流、高级统计、异步导出和完整字段权限放二期。

当前工程 `/Users/liuqiong/workspace/rental_car` 已包含 C 端微信小程序基础页面，并已整理到 `miniprogram/`，可作为 C 端首期落地基础。PC 后台和管理端小程序已建立骨架，后续需要继续规划工程实现与接口。

## 2. 产品定位与端职责

### 2.1 C 端小程序

C 端小程序是用户租车入口。用户可在开通城市选择租车时间，浏览运营中心和车行已上架且空闲的车辆，提交身份认证，认证通过后支付租金和服务费，并在订单履约中查看取还车、续租、提前还车、售后费用和结算进度。

### 2.2 山海管家管理端小程序

管理端小程序是移动履约工具，不承担完整后台配置职责。运营人员、车行和司机在移动端处理高频现场工作：查看待办、客户订单接单辅助、批发订单确认、排车、执行工单、录入车况、增加现场费用、查看本人任务和处理售后记录。

### 2.3 PC 管理后台

PC 管理后台是首期事实管理入口。后台负责城市、组织、账号、车辆、价格、规则、身份审核、订单主控、批发订单、财务结算、权限、日志和审计。管理端小程序的现场操作结果必须回流 PC 后台，PC 后台对关键业务状态拥有最终解释权。

### 2.4 三端职责边界

1. PC 后台负责配置、审核、财务、规则和审计。
2. 管理端小程序负责履约执行和移动待办。
3. C 端小程序负责用户侧交易和确认。
4. 同一业务状态以服务端为准，三端只展示和触发各自权限内的操作。
5. 高风险动作，包括退款、结算、车辆删除、账号停用、金额调整和审核驳回，首期必须在 PC 后台完成或由 PC 后台留痕确认。

## 3. 角色与权限

### 3.1 C 端角色

| 角色 | 定义 | 首期权限 |
| --- | --- | --- |
| 普通用户 | 直接通过小程序租车的用户 | 登录、浏览车辆、提交身份认证、下单、支付租金和服务费、查看订单、发起续租/提前还车、确认售后、联系客服 |
| 渠道来源用户 | 通过渠道参数进入或注册的用户 | 普通用户能力；首期仅记录渠道来源，不展示渠道工作台、价格助手和收益 |

### 3.2 管理端小程序角色

| 角色 | 定义 | 首期权限 |
| --- | --- | --- |
| 运营中心人员 | 城市级运营主体员工 | 查看客户订单待办、接单辅助、发起或跟进关联批发订单、排车、工单、车况、现场费用、售后处理 |
| 车行人员 | 车源提供和调车主体员工 | 管理本车行批发订单接拒单、车辆基础查看、工单执行、车况、现场费用、售后协同 |
| 司机 | 任务执行人 | 查看本人任务、执行工单、录入车况、增加现场费用、提交投诉或售后备注、查看本人绩效基础信息 |

托管个人首期不提供独立小程序端，只在 PC 后台和车辆资料中保留托管归属、分成比例和结算字段。

### 3.3 PC 后台角色

| 角色 | 定义 | 首期权限 |
| --- | --- | --- |
| 总部管理员 | 平台总部运营和系统管理人员 | 开通城市、运营中心、平台车型库、全局账号权限、基础统计、操作日志 |
| 运营中心管理员 | 城市级运营管理人员 | 本城市订单、车辆、车行、司机、价格、规则、身份审核、财务结算、客户、黑名单、账号权限 |
| 车行管理员 | 车行经营管理人员 | 本车行车辆、批发订单、协同工单、司机、售后协同、经营基础数据、账号管理 |
| 财务岗 | 收款、退款、结算和对账人员 | 费用明细、支付流水、退款审核、押金线下确认、批发成本、结算记录 |
| 客服岗 | 客户订单跟进和售后沟通人员 | 客户订单查看、续租提醒、提前还车跟进、售后费用跟进、客户资料查看 |
| 计调岗 | 车辆和司机调度人员 | 待排车订单、工单排班、司机闲忙、车辆排班、任务改派 |
| 审核岗 | 审核人员 | 身份认证审核、车辆审核、车行资料审核、投诉和整改审核 |

### 3.4 权限原则

1. 权限校验必须在服务端执行，前端隐藏按钮不能作为唯一控制。
2. 总部可查看全平台数据；运营中心默认只查看本城市数据；车行默认只查看本车行数据和授权协同订单。
3. 司机默认只看到本人任务；具备排单权限的班组长或管理员可查看并分配团队任务。
4. 停止合作的车行及其账号不可登录，历史订单继续保留归档。
5. C 端渠道来源只用于订单归因和二期结算准备，首期不影响用户看到的价格。
6. 字段级权限首期只覆盖敏感信息脱敏和关键金额查看，完整字段权限放二期。

## 4. 首期核心业务闭环

### 4.1 C 端租车闭环

1. 用户进入首页，系统定位或默认展示开通城市。
2. 用户选择取车时间和还车时间。
3. 系统展示运营中心和车行已上架、当前空闲、满足时间条件的车辆。
4. 用户查看车辆详情、价格日历、预定须知和费用预估。
5. 用户进入确认订单，选择取还车方式，填写地址和备注。
6. 未登录用户先完成手机号授权登录；昵称和头像由用户可选维护，不作为自动获取前提。
7. 用户提交身份认证资料，包括姓名、证件类型、证件号码、身份证照片和驾驶证照片。
8. PC 后台人工审核身份认证；审核通过后 C 端允许提交订单并支付租金和服务费。
9. 用户提交订单后，系统按城市运营中心配置的未支付锁车时长短时锁定车辆。
10. 超过锁车时长仍未支付时，未支付订单自动关闭，车辆库存释放，不进入管理端待接单，不生成关联批发订单。
11. 用户完成微信支付后，客户订单进入 PC 后台和管理端待接单。
12. 若车辆为运营中心自有或托管车辆，运营中心可直接接单并生成取还/送收车工单。
13. 若车辆为车行车辆，运营中心发起关联批发订单，由车主车行二次确认。
14. 车主车行二次确认 SLA 由城市运营中心配置；超时未确认时关联批发订单自动拒绝，并记录超时拒绝原因。
15. 车行接单后，客户订单和批发订单进入排车和履约；车行拒单或超时拒绝后，运营中心优先改派同车型车辆，无法改派则退单退款。
16. 改派产生差价时，新车价格高于原订单的差额由运营中心承担并记为让利或人工调整；新车价格低于原订单时，按新车价格重算并将差额原路退回或进入待结算退差。
17. 押金和违章押金首期后置到取车前处理，金额按车型配置，支持线下收取、免押截图人工确认和 PC 后台留痕。
18. 用户在订单中跟踪状态，必要时发起续租或提前还车。
19. 管理端完成还车、售后和费用录入后，用户在 C 端确认售后金额，PC 后台完成结算。

### 4.2 客户订单管理闭环

1. 未支付订单在城市运营中心配置的锁车时长内锁定车辆，超时未支付自动关闭并释放车辆；支付成功后的客户订单进入「待接单」。
2. 运营中心判断车辆来源和可履约性：
   - 运营中心自有或托管车辆：运营中心接单。
   - 车行车辆：发起关联批发订单并等待车主车行接单。
   - 无法履约：优先改派同车型车辆；无法改派或用户不接受改派时退单退款。
3. 接单后生成取车、还车、送车或收车工单。
4. 待排车阶段由计调员或班组长分配司机，一个工单可选择多个执行人。
5. 工单执行人通过管理端小程序录入车况、位置、时间和现场费用。
6. 出行中订单支持续租和提前还车申请，由 PC 后台或授权管理端人员审核。
7. 若客户订单有关联批发订单，续租、提前还车、任务时间和费用变化需要同步生成批发订单待处理事项。
8. 还车后如存在费用或争议，订单进入售后中；售后费用由 PC 后台或管理端录入并同步 C 端确认。
9. 用户确认售后后进入待结算；PC 后台处理退款、补收、押金确认、批发成本和利润。
10. 退单退款统一由运营中心在 PC 后台发起、财务岗审核，并优先原路退款；平台或车行原因无法履约时，全额退租金和服务费。
11. 结算完成后订单进入已完成。

### 4.3 批发调车闭环

1. 运营中心或车行可在管理端小程序或 PC 后台创建独立批发订单。
2. 客户订单选择车行车辆时，运营中心可发起关联批发订单。
3. 系统按租车时间筛选已上架、空闲、未维保、未锁车的车辆。
4. 车主车行在待接单中查看订单，可修改批发价、接单或拒绝；超出城市运营中心配置的确认 SLA 未处理时自动拒绝。
5. 批发订单默认由车主方承担送车和收车，对应工单、司机任务和履约节点归属车主方组织。
6. 接单后根据取还车方式生成取车、还车、送车或收车工单。
7. 待排车阶段由任务承担方分配司机。
8. 出行中可发起续租或提前还车，并联动任务时间和费用。
9. 售后阶段可添加维修费、停运费、车损费、违章费、补油费、补电费等费用。
10. 待结算阶段处理批发收入、批发成本、退款和利润，最后进入已完成。
11. 批发订单拒绝或超时自动拒绝后，若关联客户订单，客户订单进入待改派或待退单。

### 4.4 三端数据协同

1. PC 后台城市、服务类目、车辆、价格、规则和首页配置影响 C 端展示。
2. PC 后台车辆上下架、审核、维保、锁车和价格配置影响 C 端车辆列表、车辆详情、确认订单和管理端调车。
3. C 端身份认证、支付、续租申请、提前还车申请和售后确认回流 PC 后台。
4. PC 后台接单、退单、改派、排车、售后和结算操作同步 C 端订单状态和管理端小程序任务状态。
5. 管理端小程序工单执行、车况录入和费用新增回流 PC 后台订单、工单、费用和审计记录。
6. 财务结算、押金确认、退款、批发成本和利润以 PC 后台为最终管理入口。

## 5. C 端小程序功能需求

### 5.1 登录与用户资料

1. 支持微信手机号授权登录，登录成功后创建或更新用户基础资料。
2. 用户基础资料包括用户 ID、openid、unionid、手机号、昵称、头像、渠道来源、认证状态、黑名单状态。
3. 昵称和头像由用户可选维护，不作为微信自动获取能力的前提。
4. 未登录用户可浏览首页、车辆列表和车辆详情。
5. 提交身份认证、提交订单、查看订单、进入我的时必须登录。
6. 扫码进入小程序时可携带渠道、门店或推广参数；首期只记录来源，不参与价格计算和收益结算。

### 5.2 首页与城市

1. 默认展示用户所在城市；定位失败或城市未开通时展示默认城市，首期默认三亚。
2. 用户可查看全部开通城市并手动切换。
3. 切换城市后，车辆、价格、服务类目、热门车型和规则同步刷新。
4. 首期核心服务类目为租车；游艇、别墅、水上项目、特产等非租车类目可展示为未开通或隐藏。
5. 首页展示租车时间选择、推荐车辆、热门车型、客服电话和运营 Banner。

### 5.3 租车时间与车辆筛选

1. 默认取车时间为当前时间后可配置准备时长，默认还车时间为取车后 2 天。
2. 用户可选择取车日期时间和还车日期时间。
3. 不允许还车时间早于或等于取车时间。
4. 时间变更后，车辆可用性、价格和费用预估重新计算。
5. 支持按品牌车型、价格区间、颜色、车辆类型、能源类型、座位数、变速箱、距离或门店位置筛选。
6. 车牌号可用于运营端筛选；C 端普通用户不提供完整车牌号精确搜索。

### 5.4 车辆列表

1. C 端展示运营中心和车行所有已上架、审核通过、当前空闲、满足时间条件的车辆。
2. 维保、锁车、已下架、审核未通过、被未完成订单占用的车辆不可下单。
3. 车辆卡片展示车辆照片、品牌型号、车牌号脱敏、车辆类型、颜色、座位数、变速箱、能源类型、距离、C 端日租价。
4. 列表可展示当前空闲、3 小时内空闲、8 小时内空闲、12 小时内空闲等运营中心配置标签。
5. 价格首期统一展示 C 端价；渠道来源不改变用户看到的价格。

### 5.5 车辆详情

1. 展示车辆主图、图片轮播、品牌型号、C 端日租价、颜色、车辆类型、座位数、距离、车牌号脱敏、变速箱、能源类型。
2. 展示车辆详细信息，包括排量或年款、日限里程、取还车说明和车辆来源提示。
3. 展示价格日历，每天显示 C 端价，并区分全天可租、不可租和部分时段可租。
4. 展示预定须知，包括取车证件、取消规则、押金政策、用车规则和交接车指南。
5. 展示所选用车时间下的费用预估。
6. 点击预定进入确认订单。

### 5.6 身份认证

1. 用户提交订单和支付前必须完成身份认证。
2. 认证资料包括姓名、证件类型、证件号码、身份证照片和驾驶证照片。
3. 提交后状态为待认证审核，PC 后台人工审核。
4. 审核通过后 C 端允许提交订单并支付。
5. 审核拒绝后 C 端展示拒绝原因，用户可重新提交。
6. 身份证、驾驶证和手机号在 C 端展示时需脱敏。

### 5.7 确认订单

1. 展示预定车辆、取还车时间、租期、日限里程、取还车方式、费用明细和备注。
2. 支持两种取还车方式：
   - 上门送收：用户填写送车地址和收车地址，地址选择接入地图服务，按规则计算服务费。
   - 自取自还：展示运营中心、门店或车辆所在车行的取还地址，不生成上门服务费。
3. 修改时间后重新计算可用性、租金、服务费和预计押金信息。
4. 费用明细包括租金、上门送收服务费、优惠减免、应支付金额、车辆押金提示和违章押金提示。
5. 首期线上支付只覆盖租金和服务费；车辆押金和违章押金后置到取车前处理。
6. 押金和免押信息仅展示政策和处理状态，不作为提交订单前置支付项。

### 5.8 支付订单

1. 未完成身份认证或认证未通过时，不允许进入支付。
2. 已认证通过时发起微信支付。
3. 支付对象为租金和上门送收服务费。
4. 支付成功后订单进入待接单。
5. 支付失败、取消支付、重复支付需有明确提示和重试入口。
6. 未支付订单在城市运营中心配置的锁车时长内锁定车辆，不进入管理端待接单，不生成关联批发订单。
7. 超过锁车时长仍未支付时，订单自动关闭并释放车辆。

### 5.9 押金与免押

1. 展示车辆押金和违章押金政策。
2. 车辆押金和违章押金金额按车型配置。
3. 首期押金后置到取车前处理。
4. 支持运营人员在 PC 后台记录线下押金收取、免押截图人工确认或其他人工确认结果。
5. 芝麻信用免押首期不作为主链路强依赖；如展示二维码或说明，必须标记为人工确认或待确认状态。
6. 未完成押金或免押确认时，不允许将取车工单完成并进入租赁中。

### 5.10 订单中心

1. 支持全部、进行中、已完成、已取消分组。
2. 订单卡片展示订单编号、状态、车辆信息、取还车时间、租期、取还车方式、地址、待处理事项。
3. 订单详情展示费用明细、押金状态、支付状态、工单节点、售后费用和结算进度。
4. 用户可按状态执行操作：
   - 待认证审核：查看审核状态，重新提交资料。
   - 待支付：继续支付、取消订单。
   - 待接单：查看详情、取消订单、联系客服。
   - 待改派/待退单：查看改派车辆、差价处理结果、退款进度、联系客服。
   - 待取车：查看取车信息、联系门店或司机、查看押金确认状态。
   - 租赁中：发起续租、提前还车、查看费用明细、联系客服。
   - 待还车：查看还车信息，确认还车车况。
   - 售后中：查看售后费用，确认或反馈售后金额。
   - 待结算：查看退款、补收、押金和结算进度。
   - 已完成：查看费用明细、取还车况记录、删除本地展示记录。
   - 已取消/已退单：查看取消或退单原因和退款进度。
4. 退款状态展示为退款审核中、退款中、已退款、退款失败。
5. C 端删除订单仅为用户侧隐藏，不删除后台归档数据。

### 5.11 续租、提前还车与车况确认

1. 租赁中订单可发起续租，用户选择新的还车时间并查看新增租金预估。
2. 租赁中订单可发起提前还车；节假日来源和特殊日期退费规则由城市运营中心在 PC 后台单独配置。
3. 申请提交后进入待审核，PC 后台或授权管理端人员审核通过或拒绝。
4. 审核通过后，客户订单、关联批发订单、还车工单时间和费用同步更新或生成待处理事项。
5. 审核拒绝后 C 端展示拒绝原因。
6. 用户可查看取车和还车车况记录，包括照片、问题项、备注、时间、地点和提交人。
7. 管理端司机录入的车况应同步到 C 端可查看范围。

### 5.12 我的

1. 展示手机号、昵称、头像、角色、认证状态。
2. 支持维护昵称和头像。
3. 支持修改手机号。
4. 提供身份认证入口、订单入口、消息中心、联系客服、退出登录。
5. 渠道工作台、价格助手和渠道收益入口首期不展示。

## 6. 管理端小程序功能需求

### 6.1 首页与待办

1. 登录后展示当前组织，如三亚运营中心或某车行。
2. 展示今日待办、即时工单、待接批发订单、待排车任务、待售后事项。
3. 待办按时间线展示取车、送车、收车、还车和其他任务。
4. 当前登录人的任务高亮显示，并支持“我的任务”筛选。
5. 任务完成后从待办列表移除，并自动加载后续任务。
6. 默认加载 100 条，支持继续加载。

### 6.2 控制台入口

首期控制台按角色展示以下入口：

1. 运营中心人员：客户订单、关联批发订单、工单管理、车辆查看、司机闲忙、现场费用、售后处理。
2. 车行人员：待接批发订单、批发订单、工单管理、车辆查看、售后协同。
3. 司机：我的工单、车况录入、费用新增、投诉或售后备注、我的绩效基础信息。
4. 规则配置、财务结算、账号权限、车辆审核、价格批量设置等入口首期在 PC 后台完成。

### 6.3 调车与车辆列表

1. 管理端调车展示所在城市车辆，不允许普通车行切换至其他城市。
2. 支持输入租车日期和时间段，系统筛选空闲车辆。
3. 支持车牌号、租金、颜色、品牌车型、车辆类型、能源类型、变速箱、座位数筛选。
4. 列表展示当前空闲、3 小时内空闲、8 小时内空闲、12 小时内空闲车辆。
5. 信任车行车辆置顶并高亮，不合作车行车辆置底并置灰。
6. 列表字段包括车辆照片、车牌号、品牌型号、车辆类型、颜色、座位数、批发价或底价。
7. 运营中心和车行查看价格时，应根据角色展示批发价或授权底价。

### 6.4 批发订单管理

1. 支持独立创建批发订单。
2. 支持处理由客户订单触发的关联批发订单。
3. 批发订单状态包括待接单、待排车、待出行、出行中、待售后、待结算、已完成、已拒绝、已取消。
4. 待接单：
   - 车主车行可查看用车方、车辆、时间、取还方式、费用和备注。
   - 车主车行可修改批发价和取还车费用。
   - 车主车行可接单或拒绝。
   - 车主车行确认 SLA 由城市运营中心配置，超时未确认时自动拒绝。
   - 拒绝或超时自动拒绝关联批发订单时必须记录原因，并同步客户订单进入待改派或待退单。
5. 待排车：
   - 批发订单默认由车主方承担送车和收车，相关工单、司机任务和履约节点归属车主方组织。
   - 双方按任务承担方分配司机。
   - 一个工单可选择多个司机。
6. 出行中：
   - 支持续租和提前还车。
   - 时间、费用和还车工单变更需同步关联客户订单或生成待处理事项。
7. 待售后：
   - 支持添加维修费、停运费、车损费、违章费、补油费、补电费和其他费用。
   - 费用结清或确认后进入待结算。
8. 待结算：
   - 支持查看批发收入、批发成本、退款和利润。
   - 结算完成后进入已完成。

### 6.5 客户订单移动处理

1. 管理端小程序可查看客户订单待接单、待排车、待出行、出行中、待售后、待结算、已完成、已退单、已取消。
2. 运营中心人员可在移动端辅助接单、退单、改派和发起关联批发订单；关键退款和最终结算以 PC 后台为准。
3. 待接单时展示车辆来源：
   - 运营中心自有或托管车辆：可接单并生成工单。
   - 车行车辆：发起或查看关联批发订单状态。
4. 车行拒绝或超时自动拒绝关联批发订单后，客户订单进入待改派或待退单；运营中心优先改派同车型车辆，无法改派或用户不接受改派时由 PC 后台退单退款。
5. 待排车时，计调员或班组长给取车、还车、送车、收车工单分配司机。
6. 出行中支持查看续租提醒、提前还车申请和还车提醒。
7. 待售后支持录入费用和凭证，并同步 C 端用户确认。
8. 待结算仅展示结算信息和待处理事项，实际退款、押金确认和利润结算在 PC 后台完成。

### 6.6 工单任务管理

1. 工单类型包括取车、送车、收车、还车、取借车、还借车、送借车、收借车、维修、保养、采购、其他。
2. 工单状态包括待排单、待开始、进行中、已完成、已取消。
3. 待开始操作：
   - 排单：有权限人员分配司机。
   - 转单：司机申请或管理员改派给其他司机。
   - 已出发：工单状态变为进行中。
4. 进行中操作：
   - 车况录入：上传车辆照片、油量或电量、里程、外观问题、当前位置和时间。
   - 增加费用：补油费、补电费、超时费、停车费、其他现场费用。
   - 转售后：填写售后备注，任务完成并关联售后事项。
   - 客人投诉：记录投诉内容和照片。
   - 完成任务：任务进入已完成。
5. 取车完成前必须确认押金或免押状态。
6. 首期不支持离线提交车况照片；无网络时提示恢复网络后再提交。
7. 工单完成后自动回写订单履约节点。
8. 工单列表支持全部、今天、明天、后天、状态、类型、执行人和日期筛选。

### 6.7 车辆、司机和现场费用

1. 管理端小程序可查看车辆基础资料、上架状态、租用状态、维保状态和价格概要。
2. 车辆新增、审核、批量价格、上下架规则配置首期在 PC 后台完成。
3. 司机可查看本人任务和本人绩效基础信息。
4. 运营中心人员可查看司机闲忙状态并用于排车。
5. 现场费用由工单或订单进入，必须关联费用类型、金额、凭证、备注和创建人。
6. 关键金额调整需填写原因并记录日志。

## 7. PC 管理后台功能需求

### 7.1 工作台

1. 登录后根据角色展示总部、运营中心或车行工作台。
2. 总部工作台展示开通城市数、运营中心数、车行数、车辆数、用户数、订单数、营业额、待处理审核和异常订单。
3. 运营中心工作台展示本城市待认证审核、待接单、待排车、待售后、待结算、今日已收、今日待退、车辆空闲率、司机空闲情况。
4. 车行工作台展示本车行车辆数、空闲车辆、待接批发订单、待排车任务、售后待处理和结算待处理。
5. 支持待办卡片跳转到对应业务列表，并保留筛选条件。
6. 首期只做核心指标和待办，不做高级统计分析。

### 7.2 平台与城市配置

1. 支持新增、编辑、停用开通城市。
2. 城市字段包括城市名称、城市编码、运营状态、默认运营中心、服务类目、联系人、联系电话。
3. 支持设置默认兜底城市，首期默认三亚。
4. 支持新增、编辑、停用运营中心。
5. 支持配置城市开通服务类目，首期核心类目为租车。
6. 支持配置 C 端首页 Banner、热门车型、推荐车辆、价格区间、客服电话、营业时间、默认取还车门店、地图围栏和服务半径。
7. 支持按城市运营中心配置未支付订单锁车时长、车主车行二次确认 SLA、节假日来源和提前还车退费规则。
8. 支持按城市运营中心配置上门送收服务费规则，包括地图服务、服务半径、超区费和夜间费。
9. 停用城市后，C 端不允许切换到该城市，也不展示该城市车辆。

### 7.3 组织、账号与权限

1. 支持组织结构：总部、运营中心、车行、部门、岗位。
2. 支持账号新增、编辑、停用、重置密码、锁定、解锁。
3. 支持角色管理：总部角色、运营中心角色、车行角色、司机角色。
4. 首期权限控制到菜单、页面、按钮和基础数据范围。
5. 完整字段权限、可配置审批流和复杂数据权限放二期。
6. 停止合作的车行及其账号不可登录。

### 7.4 车型、车辆与库存

1. 总部维护平台车型库，包括品牌、车系、车型、年款、车辆类型、能源类型、座位数、变速箱。
2. 支持车辆列表，按已上架、已下架、审核中、维保中、租赁中、锁车状态分组。
3. 支持按城市、车行、品牌车型、车牌号、来源、上下架状态、租用状态、审核状态筛选。
4. 车辆来源必须显式区分：运营中心自有、托管车辆、车行车辆。
5. 新增车辆字段包括品牌、车型、年款、车牌号、颜色、车辆类型、座位数、能源类型、变速箱、公里数、日限里程、车辆照片、来源、所属车行、托管个人、审核状态、上下架状态、维保状态。
6. 托管车辆首期需记录托管个人、分成比例和结算备注，但不提供托管个人独立端。
7. 车辆上架前必须满足审核通过、价格有效、取还车地址有效、车辆照片完整。
8. 已有关联未完成订单的车辆不可删除，可下架、维保或锁车。
9. 系统根据订单、批发订单、工单、维保、上下架和人工锁车计算可租状态。
10. C 端和调车列表只展示审核通过、已上架、未维保、未锁车、未被未完成订单占用的车辆。

### 7.5 价格管理

1. 首期按车辆日期维护底价、C 端价、批发价、是否可租和可租时段。
2. 底价用于成本和车行内部参考。
3. C 端价用于 C 端车辆列表、详情、确认订单和租金计算。
4. 批发价用于调车列表、批发订单和关联批发成本。
5. 支持按车辆设置价格日历，单次批量设置最长 90 天。
6. 支持车型最低价，车行设置底价时不得低于该最低价。
7. 支持价格缺失日期提醒和价格冲突提醒。
8. 渠道价、渠道加价上限、渠道收益和异地运营中心价首期不参与计算，保留字段扩展位到二期。
9. 用户提交订单时应锁定订单价格，后续价格变更不影响已支付订单。

### 7.6 客户与身份认证

1. 支持客户列表，按手机号、姓名、认证状态、渠道来源、黑名单状态和注册时间筛选。
2. 客户详情展示基础资料、身份认证资料、驾驶证资料、订单记录、售后记录和操作日志。
3. 支持身份认证人工审核。
4. 审核通过后 C 端可继续提交订单和支付。
5. 审核拒绝需填写原因，并同步给 C 端用户。
6. 支持黑名单管理；黑名单用户首期禁止进入小程序。
7. 敏感信息需脱敏展示，查看完整证件资料需要具备权限并记录日志。

### 7.7 客户订单管理

1. 支持客户订单列表，按待认证审核、待支付、待接单、待改派/待退单、待排车、待出行、租赁中、待还车、售后中、待结算、已完成、已取消、已退单分组。
2. 支持按订单编号、客户姓名、手机号、车牌号、车辆来源、订单来源、渠道来源、支付状态、城市、车行、时间范围筛选。
3. 订单详情展示客户信息、身份认证、车辆信息、渠道来源、取还车时间、取还车方式、地址、费用明细、支付流水、押金状态、关联工单、关联批发订单和操作日志。
4. 待支付订单只在 C 端支付链路和 PC 查看中存在，不进入管理端待接单。
5. 待接单订单支持接单、退单、改派车辆、发起关联批发订单。
6. 车辆为车行车辆时，运营中心发起关联批发订单；车行接单后客户订单继续履约，车行拒单后客户订单进入待改派/待退单。
7. 改派优先选择同车型车辆；改派后 C 端展示新车辆信息、差价处理结果和客服入口。
8. 改派导致价格变化时，新车价格高于原订单的差额由运营中心承担并记为让利或人工调整；新车价格低于原订单时，按新车价格重算并将差额原路退回或进入待结算退差。
9. 待排车订单支持分配取车、还车、送车、收车工单。
10. 出行中订单支持续租申请审核、提前还车申请审核、还车提醒和费用调整。
11. 售后中订单支持添加售后费用、上传凭证、提交客户确认和售后完成。
12. 待结算订单支持收款、退款、押金确认、批发成本、利润核算和结算完成。
13. 支持手动订单录入，包括订单来源、客户资料、驾驶证照片、车辆来源、取还车地址、费用明细和备注。
14. 客户订单关键操作必须记录操作日志。

### 7.8 批发订单管理

1. 支持批发订单列表，按待接单、待排车、待出行、出行中、待售后、待结算、已完成、已拒绝、已取消分组。
2. 支持区分独立批发订单和客户订单关联批发订单。
3. 支持区分借入订单和借出订单。
4. 待接单批发订单支持查看用车方合作状态、修改批发价、接单、拒绝。
5. 拒绝或超时自动拒绝关联批发订单必须记录原因，并同步客户订单进入待改派/待退单。
6. 车主车行二次确认 SLA 由城市运营中心配置，超时未确认时关联批发订单自动拒绝。
7. 批发订单默认由车主方承担送车和收车，相关工单、司机任务和履约节点按车主方组织生成。
8. 待排车批发订单支持按任务承担方分配司机，一个工单可分配多个司机。
9. 出行中批发订单支持续租、提前还车、费用调整和任务时间调整。
10. 售后阶段支持添加维修费、停运费、车损费、违章费、补油费、补电费和其他费用。
11. 结算阶段支持计算批发成本、批发收入、退款和利润。
12. 客户订单关联批发订单后，时间、任务、费用变化需同步生成待处理提醒，避免客户订单与批发订单数据不一致。

### 7.9 工单与排班

1. 支持工单列表，按待排单、待开始、进行中、已完成、已取消分组。
2. 支持工单类型：取车、送车、收车、还车、取借车、还借车、送借车、收借车、维修、保养、采购、其他。
3. 支持按工单类型、执行人、车辆、订单编号、任务时间和状态筛选。
4. 支持排单、改派、转单和取消工单。
5. 支持查看司机闲忙日历和车辆排班日历。
6. 工单详情展示关联订单、关联批发订单、任务地点、任务时间、执行人、车况记录、费用记录、投诉记录和定位信息。
7. PC 后台可代司机录入或修正车况记录，但需记录操作原因和日志。
8. 工单完成后自动回写订单履约节点。

### 7.10 财务、押金与结算

1. 财务工作台展示今日已收、今日待收、今日待退、押金待确认、售后待收、批发待结。
2. 支持查看订单费用明细、支付流水、退款流水、押金确认记录和人工调整记录。
3. 首期费用类型包括租金、上门送收服务费、车辆押金、违章押金、批发成本、售后费用、退款、人工调整。
4. 租金和服务费通过微信支付。
5. 车辆押金和违章押金金额按车型配置，首期支持线下收取、免押截图人工确认和人工备注。
6. 未确认押金或免押状态时，取车工单不可完成。
7. 退单退款统一由运营中心在 PC 后台发起、财务岗审核，并优先原路退款。
8. 退款类型包括退单退款、押金退款、提前还车退款、售后退款、其他退款。
9. 退款状态包括退款审核中、退款中、已退款、退款失败。
10. 平台或车行原因导致无法履约时，全额退租金和服务费；用户主动取消、提前还车、售后退款按城市运营中心配置的取消和退费规则计算。
11. 支持待结算订单的批发成本、司机绩效基础金额和利润核算。
12. 渠道收益结算、托管收益线上化和复杂对账导出放二期。

### 7.11 售后、投诉与报销

1. 售后可从订单、工单或投诉进入。
2. 支持登记售后原因、责任方、费用类型、金额、凭证和备注。
3. 售后费用需同步 C 端用户确认。
4. 用户确认后进入待结算；用户拒绝时生成待跟进记录。
5. 投诉首期支持车辆投诉和员工投诉基础记录。
6. 车辆投诉发起整改时可自动下架车辆，整改完成后由运营中心审核重新上架。
7. 报销审批首期只做基础记录和审批状态；完整报销流程可放二期。

### 7.12 通知、消息与日志

1. 支持消息中心展示订单、工单、审批、财务、投诉、系统公告等消息。
2. 支持小程序订阅消息或站内信通知订单状态、续租提醒、待办提醒、审核结果、售后确认。
3. 短信模板管理放二期；首期短信仅作为验证码或重要通知接口预留。
4. 操作日志记录登录、数据新增、修改、删除、审核、支付、退款、结算、导出、金额调整和权限变更。
5. 日志字段包括操作人、角色、组织、操作时间、来源 IP 或设备信息、操作前后内容。

### 7.13 基础数据导出

1. 首期 PC 后台采用固定导出模板，不支持自定义字段模板。
2. 固定导出模板包括客户订单、批发订单、车辆、客户/认证、工单、费用/支付/退款、司机绩效基础数据。
3. 导出操作需受数据权限控制，并记录导出人、导出时间、导出范围和来源 IP。
4. 异步导出、自定义字段模板和审计报表放二期。

## 8. 关键状态定义

### 8.1 C 端客户订单状态

| 状态 | 触发条件 | 用户侧可操作 |
| --- | --- | --- |

| 待支付 | 身份认证通过，订单待支付租金和服务费 | 支付、取消订单 |
| 已关闭 | 待支付订单超过城市锁车时长未支付 | 查看关闭原因、重新下单 |
| 待接单 | 支付成功，运营中心未接单 | 查看详情、取消订单、联系客服 |
| 待改派/待退单 | 车行拒绝、超时自动拒绝关联批发订单或车辆无法履约 | 查看改派车辆、差价处理、退款进度、联系客服 |
| 待取车 | 已接单并完成排车，取车或送车未完成 | 查看取车信息、联系门店或司机、查看押金状态 |
| 租赁中 | 取车完成且押金或免押已确认 | 续租、提前还车、联系客服 |
| 待还车 | 到达还车阶段或用户已发起还车 | 查看还车信息、确认车况 |
| 售后中 | 还车后存在费用或争议 | 查看售后费用、确认或反馈金额 |
| 待结算 | 售后完成，退款、补收或押金结算中 | 查看结算进度 |
| 已完成 | 全部费用、押金和结算完成 | 查看详情、删除本地展示记录 |
| 已取消/已退单 | 用户取消、管理端退单、车行拒单后无法改派或超时取消 | 查看原因和退款进度 |

| 状态 | 管理动作 |
| --- | --- |

| 待还车 | 跟进还车工单和车况确认 |
| 售后中 | 添加费用、提交客户确认、完成售后 |
| 待结算 | 退款、收款、押金确认、批发成本、利润、司机绩效基础金额 |

| 已取消/已退单 | 查看取消、退单和退款原因 |

## 11. 非功能需求

### 11.1 性能

1. C 端首页首屏优先展示城市、时间选择和首批车辆。
2. 车辆列表分页加载，图片使用缩略图和懒加载。
3. 管理端待办默认加载 100 条，继续加载时保持滚动体验。
4. PC 后台列表默认分页加载。
5. PC 工作台先加载核心指标和待办，再加载明细。

### 11.2 安全

1. 所有用户输入需做服务端校验。
2. 身份证、驾驶证、手机号等敏感信息需脱敏展示。
3. 支付、退款、押金确认、结算和金额调整必须记录操作日志。
4. 权限校验必须在服务端执行。
5. PC 后台需支持账号停用、密码重置、登录失败锁定和数据导出权限控制。
6. 总部、运营中心、车行的数据范围必须服务端隔离。
7. 不得将密钥、凭证和支付配置硬编码进前端或源码仓库。

### 11.3 可审计

1. 订单接单、退单、拒单、改派、排车、费用新增、退款、收款、押金确认、结算、价格修改、车辆上下架、审核和投诉整改需有日志。
2. 日志字段包括操作人、角色、组织、操作时间、操作前后内容、来源 IP 或设备信息。
3. 金额调整、权限变更、账号停用、导出操作需记录独立审计日志。

### 11.4 兼容性

1. C 端和管理端以微信小程序为最终运行环境。
2. H5 转出的页面仅作为 UI 和交互参考，研发需按微信小程序组件、生命周期和接口重新实现。
3. 当前 C 端已有微信小程序工程可作为首期落地基础。
4. PC 后台以 Web 管理后台为最终运行环境，需兼容主流桌面浏览器。
5. PC 后台与两个小程序共用核心业务数据和状态，不应形成独立孤岛。

5. 投诉整改完整闭环。
6. 高级统计分析和经营报表。
7. 完整字段权限、复杂数据权限和可配置审批流。
8. 短信模板和通知公告管理。
9. 批量导入、异步导出、自定义字段模板和审计报表。
10. 芝麻信用免押自动回调和线上化核销。

## 13. 已确认决策与默认规则

1. 未支付订单需要短时锁车，锁车时长由城市运营中心单独配置；超过锁车时长仍未支付时，订单自动关闭并释放车辆。
2. C 端用户选择车行车辆并支付成功后，车主车行二次确认 SLA 由城市运营中心单独配置；超时未确认时关联批发订单自动拒绝。
3. 改派同级车辆首期定义为同车型车辆。
4. 改派产生差价时：
   - 新车价格高于原订单：用户不补差，差额记为运营中心让利或人工调整。
   - 新车价格低于原订单：按新车价格重算，差额原路退回或进入待结算退差。
   - 改派后 C 端展示新车辆信息、差价处理结果和客服入口。
   - 用户不接受改派时，进入退单退款流程。
5. 退单退款统一由运营中心在 PC 后台发起，财务岗审核后优先原路退款；平台或车行原因无法履约时，全额退租金和服务费。
6. 用户主动取消、提前还车、售后退款按城市运营中心配置的取消和退费规则计算。
7. 退款状态统一展示为退款审核中、退款中、已退款、退款失败。
8. 车辆押金和违章押金金额按车型配置。
9. 人工免押确认凭证首期为免押截图。
10. 节假日来源和提前还车不退费规则由城市运营中心单独配置。
11. 上门送收服务费规则由城市运营中心单独配置，配置项包括地图服务、服务半径、超区费和夜间费。
12. 批发订单默认由车主方承担送车和收车，相关工单、司机任务和履约节点按车主方组织生成。
13. 黑名单用户首期禁止进入小程序。
14. 管理端首期不支持离线提交车况照片；无网络时提示恢复网络后再提交。
15. PC 后台首期采用固定导出模板，不支持自定义字段模板。
16. 首期固定导出模板包括客户订单、批发订单、车辆、客户/认证、工单、费用/支付/退款、司机绩效基础数据。

## 14. 验收口径

1. C 端用户可从首页完成选车、看详情、提交身份认证、审核通过后支付租金和服务费，并查看订单。
2. 未完成或未通过身份认证的用户不能进入支付。
3. 未支付订单在城市配置锁车时长内锁定车辆，超时后自动关闭并释放车辆。
4. 未支付订单不进入管理端待接单，不生成关联批发订单。
5. 用户选择运营中心自有或托管车辆并支付后，运营中心可接单并生成取还或送收车工单。
6. 用户选择车行车辆并支付后，运营中心可发起关联批发订单，车主车行可接单或拒绝。
7. 车主车行超出城市配置确认 SLA 未处理时，关联批发订单自动拒绝并记录原因。
8. 车行拒绝或超时自动拒绝关联批发订单后，运营中心可改派同车型车辆；无法改派或用户不接受改派时可退单退款。
9. 改派至高价车辆时用户不补差，差额记为运营中心让利或人工调整；改派至低价车辆时按规则退差。
10. 平台或车行原因无法履约时，PC 后台可由运营中心发起全额退款，财务岗审核后 C 端展示退款进度。
11. 车辆价格、上架状态、审核状态、维保状态、锁车状态和订单占用能正确影响 C 端和调车列表。
12. 价格变更后，C 端列表、详情、确认订单和批发订单价格口径一致；已支付订单价格不被后续变更影响。
13. 车型押金配置能影响 C 端押金提示和取车前押金确认。
14. 押金线下确认或上传免押截图并人工确认后，取车工单才能完成并进入租赁中。
15. 城市运营中心可配置节假日来源、提前还车退费规则、上门送收服务半径、超区费和夜间费。
16. 批发订单送车、收车工单归属车主方组织。
17. 黑名单用户无法进入小程序。
18. 管理端无网络时不能离线提交车况照片。
19. 司机工单执行结果能回写订单和车况记录。
20. 售后费用由 PC 后台或管理端录入后，C 端用户可查看并确认。
21. PC 后台可完成客户订单从接单、改派、排车、工单、售后到结算的完整管理闭环。
22. PC 后台可完成独立批发订单和客户订单关联批发订单闭环。
23. PC 后台可按固定模板导出客户订单、批发订单、车辆、客户/认证、工单、费用/支付/退款、司机绩效基础数据。
24. 订单状态在 C 端、管理端小程序和 PC 后台保持一致。
25. 不同角色只能看到和操作其权限范围内的数据。
26. 租金、服务费、押金确认、退款、批发成本、售后费用和人工调整均有明细和操作日志。
27. 渠道收益、渠道价格助手、托管个人完整端、复杂审批流和高级统计不出现在首期验收范围内。
